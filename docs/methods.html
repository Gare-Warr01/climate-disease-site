Overview of the Methods

This project was built to be transparent, reproducible, and easy to follow.
Every graph, table, and comparison on this website comes from a clear, step-by-step workflow in R that someone else could recreate.

Data Sources

To explore how climate conditions relate to mosquito-borne disease, this project draws from two major groups of datasets.

Climate Data

  • Sourced from NOAA climate summaries.
  • Includes temperature, precipitation, and drought indices such as PDSI.
  • Aggregated to the state–year level for consistent national comparison.

Disease Data

  • Built from human case data for West Nile virus (WNV) from CDC public health surveillance.
  • Includes reported cases per state and year.
  • Focuses on long-term, national patterns rather than short-term outbreaks.

All datasets were used strictly for educational purposes in GEOG 3000.

Data Cleaning & Preparation

All data cleaning was performed in R using the tidyverse.

Key steps included:

  • Importing CSVs with readr
  • Standardizing column names for readability
  • Filtering to valid states and meaningful years
  • Checking for missing or inconsistent values
  • Removing invalid observations (e.g., negative case counts)
  • Creating tidy, analysis-ready tables

One of the biggest lessons here: real-world data is imperfect, and careful cleaning is essential for honest analysis.

Combining Climate & Disease Data

To make meaningful comparisons, the datasets needed to align.

This involved:

  • Standardizing state abbreviations
  • Ensuring matching year formats
  • Creating a shared state–year joining key
  • Using dplyr::left_join() to merge climate indicators with WNV case counts

The unified dataset supports the visuals throughout:

  • Climate Trends
  • Disease Trends
  • Climate–Disease Comparison
  • Utah Focus

The goal was not prediction, but to create a clean structure where visual patterns could be explored with confidence.

Visualization & Analysis in R

All graphs were created using ggplot2, chosen for its clear, consistent visual style.

The project uses:

  • Line charts for long-term climate and disease trends
  • Faceted plots for comparing states
  • Paired or overlayed visuals to explore climate–disease relationships
  • Clean color palettes for accessibility

This project emphasizes:

  • Visual storytelling
  • Exploratory data analysis
  • Clear communication

Readers should be able to interpret trends without needing advanced statistical training.

Reproducibility & Workflow

A major priority was ensuring this project was reproducible.

This was supported by:

  • Building each page in R Markdown
  • Keeping explanations and code together
  • Using self-updating graphics that knit automatically
  • Hosting the entire project on GitHub Pages
  • Maintaining a clear and traceable workflow

Reproducibility is not just a coding goal — it’s a commitment to scientific clarity and transparency.

Analytical Choices & Limitations

This project uses descriptive and visual analysis rather than predictive modeling.
This choice keeps the results accessible and avoids overstating what the datasets can support.

Limitations include:

  • Differences in reporting practices between states
  • Possible under-reporting of human WNV cases
  • Occasional gaps or inconsistencies in climate data
  • State-level aggregation that cannot capture city-level variation

These limitations do not prevent meaningful exploration — they simply frame how the trends should be interpreted.

Software & Versioning

To support reproducibility, all work was completed using:

  • R (version 4.x)
  • RStudio
  • tidyverse
  • ggplot2
  • knitr and rmarkdown for site rendering

Using consistent versions ensures that graphs and results remain stable over time.
